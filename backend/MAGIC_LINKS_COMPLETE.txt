🔒 SECURITY IMPLEMENTED

✅ Email Enumeration Protection
   - Never reveal if email exists/doesn't exist
   - Same response: "If that email exists, a link was sent."

✅ Single-Use Tokens
   - `used` column in magic_links table
   - After verification: UPDATE magic_links SET used=true
   - Prevents replay attacks

✅ Token Expiry
   - 15-minute window on all magic links
   - Checked: if (new Date(ml.expires_at) < new Date())

✅ Transactional Updates
   - Multi-step operations wrapped in BEGIN/COMMIT/ROLLBACK
   - Prevents partial updates on connection failure

✅ No JWT After Password Reset
   - Password reset returns 200 without token
   - Forces re-login with new password
   - Invalidates all other magic links for security

✅ Database Constraints
   - magic_links.token: UNIQUE (no duplicates)
   - magic_links.user_id: Foreign key with CASCADE delete
   - magic_links.type: ENUM (email_verification, passwordless_login, password_reset)
   - All queries use parameterized statements (no SQL injection)

✅ Rate Limiting
   - 5 requests per 15 minutes on password/login endpoints
   - Prevents brute force attacks
